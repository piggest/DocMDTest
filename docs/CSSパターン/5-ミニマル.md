# 設計思想

ソフトウェアの設計とは、制約の中で美しさを見つける営みである。

---

## 単純さについて

### 削ることの意味

設計において「加える」ことは容易い。「削る」ことに技術と勇気がいる。

優れた設計は、複雑さを隠すのではなく、複雑さを消す。ユーザーは本来の目的に集中できるようになる。これが設計の理想形だ。

**単純さの三原則**

- 不要な要素を排除する
- 必要な要素を明確にする
- 要素同士の関係を整理する

### 一貫性の力

一貫したシステムは、予測可能だ。予測可能なシステムは、信頼できる。

## 構造と形式

### 良い設計の特性

| 特性 | 説明 |
|------|------|
| 明快さ | 意図が一目でわかる |
| 柔軟性 | 変化に対応できる |
| 堅牢性 | 予期しない入力に耐える |
| 効率性 | 最小の手段で最大の効果 |

### 設計の実践

```python
# 良い設計: 一つの関数は一つのことをする
def parse_config(path: str) -> dict:
    """設定ファイルを読み込み、辞書として返す。"""
    with open(path) as f:
        return json.load(f)
```

`parse_config` は名前の通りのことだけをする。副作用なし。テスト容易。

---

> 設計は問題解決ではない。問題発見だ。

---

設計は一度完成したら終わりではない。システムが成長するにつれ、設計も進化する。その変化を拒まず、しかし核心を守る。それが長く生きるソフトウェアの条件だ。

1. 現状を理解する
2. 問題を定義する
3. 解を探索する
4. 実装し、検証する
5. 繰り返す

---

*設計は技術ではなく、態度だ。*
