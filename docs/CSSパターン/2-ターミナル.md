# サーバー運用手順

本ドキュメントはプロダクション環境のサーバー運用・障害対応手順をまとめたものです。オンコール担当者は必ず一読してください。

---

## 初期セットアップ

### SSH接続

踏み台サーバー経由でプロダクション環境に接続します。

```bash
# 踏み台サーバーへ接続
ssh -i ~/.ssh/bastion.pem ec2-user@bastion.example.com

# プロダクションサーバーへ転送
ssh -A prod-server-01
```

### 環境変数の確認

```bash
# 必須環境変数の確認
echo $DATABASE_URL
echo $REDIS_URL
echo $APP_ENV  # production であることを確認
```

## デプロイ手順

1. リリースブランチをプル
2. マイグレーションの確認
3. ローリングデプロイ実行
4. ヘルスチェック確認
5. ログ監視（5分間）

**重要**: デプロイ前に必ずステージング環境で動作確認を行うこと。

## 監視コマンド一覧

| コマンド | 用途 | 備考 |
|---------|------|------|
| `htop` | リソース監視 | CPU/メモリ |
| `tail -f /var/log/app.log` | リアルタイムログ | エラー検出 |
| `docker ps` | コンテナ状態確認 | 全コンテナ |
| `netstat -tlnp` | ポート確認 | 開放ポート一覧 |
| `df -h` | ディスク使用量 | 90%超で警告 |

## 障害対応フロー

> **P0インシデント**: サービス完全停止。即座にオンコール全員へ通知し、インシデントチャンネルを開設すること。

### ログ収集

```bash
# エラーログの抽出（直近100件）
grep -E "ERROR|FATAL" /var/log/app.log | tail -100

# 特定時刻以降のログ
journalctl --since "2024-01-15 14:00:00" -u app.service
```

### ロールバック手順

`kubectl rollout undo deployment/app-deployment` を実行後、ステータスを確認してください。
