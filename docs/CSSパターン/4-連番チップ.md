# 連番チップ / プロダクト設計ガイドライン

このドキュメントは、社内プロダクト開発における設計判断の基準と意思決定プロセスをまとめたものです。

## 概要

プロダクト設計は、ユーザー体験・技術的制約・ビジネス要件の3つを統合して進める作業です。本ガイドラインでは、各観点を整理する手順と、レビュー基準を定めます。

### 対象読者

エンジニア、デザイナー、PM。新規プロダクトの立ち上げ時、および既存プロダクトの大規模リニューアル時に参照してください。

## 設計プロセス

設計は以下のフェーズで進めます。

1. リサーチ・要件定義
2. コンセプト検討
3. プロトタイピング
4. 詳細設計
5. レビュー・合意形成

### 各フェーズの所要時間

| フェーズ | 標準的期間 | 主な成果物 |
|----------|-----------|-----------|
| リサーチ | 1〜2週間 | 調査レポート |
| コンセプト | 1週間 | コンセプトシート |
| プロトタイプ | 2〜3週間 | 動くデモ |
| 詳細設計 | 2週間 | 設計書 |
| レビュー | 1週間 | 合意議事録 |

## 判断基準

> 設計の良し悪しは、3年後の自分が読んで理解できるかで決まる。

迷った時は次の優先順位で判断します。

- **ユーザー体験** > 技術的美しさ
- **シンプルさ** > 多機能
- **既存パターンの踏襲** > 独自設計

### コードサンプル

設定ファイルの例:

```yaml
project:
  name: example
  version: 1.0.0
  features:
    - search
    - export
    - sync
```

## まとめ

良い設計は議論から生まれます。一人で抱え込まず、早い段階でレビューを受けてください。
